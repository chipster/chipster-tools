Convert mothur files into phyloseq object

Description

Imports data into the phyloseq format using the import_mothur() function in R, and saves the output as an Rda file.

Parameters

  • Phenodata variable with unique IDs for each community profile

Specifying this variable is required to correctly import phenodata variables into the phyloseq object.

Details

This tool requires three input files:

  • Shared file representing OTU observations in samples (file.opti_mcc.shared)
  • Constaxonomy file with consensus taxonomies for OTUs (file.opti_mcc.0.0X.cons.taxonomy)
  • Phenodata file (used to assign samples for statistical analysis)
As part of the import process, initial taxonomic rankings are renamed for clarity (Domain_Kingdom, Phylum, Class, Order, Family, Genus, Species). The input data may include taxonomic assignments at six levels (Domain_Kingdom to Genus) or seven levels (Domain_Kingdom to Species), depending on the taxonomic reference file used. Assignments under the label Domain_Kingdom may consist of either domain- or kingdom-level classifications, depending on the reference employed. Where phyloseq object names contain "-" symbols, these are replaced with "." to ensure matching sample names between the phyloseq object and the phenodata file.

Output

The tool produces the following output files:

  • Phyloseq object saved as an Rda file (ps.Rda)
  • Text summary of imported phyloseq object, sample names and variables (ps_imported.txt)
