Hypergeometric test for GO

Description

Given a list of Ensembl or Entrez Gene identifiers, this tools performs a statistical test for enrichment of GO terms.

Parameters

  • Genome (human, mouse, rat, fruitfly, arabidopsis) [human]
  • Identifier column
  • Ontology (all, biological process, molecular function, cellular component) [biological process]
  • P-value threshold (0-1) [0.05]
  • Minimum population (1-1000000) [5]
  • Conditional testing (yes, no) [yes]
  • P-value adjustment method (none, BH, BY) [none]
  • Search for over or under-represented genes (under, over) [over]

Details

This tool takes a list of Entrez Gene or Ensembl identifiers. You need to indicate in which column the identifiers are. If they are in the rownames column (the first column, no title), select untitled.

A hypergeometric test is used to check for over-represented (or under-represented) GO categories relative to the whole genome. The analysis can be restricted to GO terms which have a minimum, user-specified number of genes mapped to them (by default 5).

In order to avoid the identification of directly related GO terms with considerable overlap of genes, this tool uses by default conditional testing: First the leaves of the graph (those terms with no child terms) are tested. All genes annotated at significant children are then removed before testing the parent terms. This continues until all terms have been tested. It is also possible to run the tests in an unconditional fashion by setting the appropriate parameter. In that case it is also possible to apply multiple testing correction.

Output

Output is a text file and an HTML file with a list of significant GO categories.
