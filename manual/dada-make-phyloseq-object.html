Make a Phyloseq object

Description

Given a sequence_table saved as a .Rda object and a taxonomy_table saved as a .Rda object this tool creates an new Phyloseq object and saves it as an .Rda file. It will create also an Phenodata file, which can be used to save sample information and then merge it to the Phyloseq object with the tool "Merge phenodata to the phyloseq object".

Parameters

  • None

Details

This tool creates a new Phyloseq object. To make a new phyloseq object, it needs an otu_table (ASV table created with "make contigs and remove chimeras") and a taxonomy_table (created with "assign taxonomy") saved as a .Rda file.

s part of the import process, initial taxonomic rankings are renamed for clarity (Domain_Kingdom, Phylum, Class, Order, Family, Genus, Species). The input data may include taxonomic assignments at six levels (Domain_Kingdom to Genus) or seven levels (Domain_Kingdom to Species), depending on if addSpecies command was used or not. Assignments under the label Domain_Kingdom may consist of either domain- or kingdom-level classifications, depending on the reference employed.

This tool creates also a metadata file called Phenodata which user can use to specify sample information for the data analyses process with the phyloseq object. To combine the sample information from the phenodata file, you should run the tool "Merge phenodata to the phyloseq object"

Output

The analysis output consists of the following:
  • ps_nophe.Rda: A Phyloseq object without phenodata.
  • ps_imported.txt: Text summary of the phyloseq object.
