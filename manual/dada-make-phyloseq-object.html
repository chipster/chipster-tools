Make a Phyloseq object

Description

Given a sequence_table and a taxonomy_table saved as a .Rda object, this tool creates a new phyloseq object and saves it as a .Rda file. It will create also a phenodata table, which can be used to specify sample information and sample groups. It can be merged to the phyloseq object with the tool "/Merge phenodata to the phyloseq object".

Parameters

  • None

Details

This tool creates a new phyloseq object. To make a new phyloseq object, it needs an otu_table (ASV table created with "make contigs and remove chimeras") and a taxonomy_table (created with "assign taxonomy") saved as a .Rda file.

For visualization purposes, the names of the ASV sequences are changed from the full DNA sequences to ASV1, ASV2...
The full DNA sequences of the ASVs are stored in the refseq() slot of the phyloseq object. That way, the short ASV names will appear in tables and plots but it's still possible to find the DNA sequences corresponding to each ASV with the tool "Extract information from the phyloseq object".
That way it's still possible for example to merge the found ASVs with other datasets or index into reference databases.

As part of the import process, initial taxonomic rankings are renamed for clarity (Domain_Kingdom, Phylum, Class, Order, Family, Genus, Species). The input data may include taxonomic assignments at six levels (Domain_Kingdom to Genus) or seven levels (Domain_Kingdom to Species). Assignments under the label Domain_Kingdom may consist of either domain- or kingdom-level classifications, depending on the reference file employed.

This tool creates also a metadata file called phenodata.tsv which the user can use to specify sample information for the data analysing steps. To combine the sample information from the phenodata table and the phyloseq object, you should run the tool "Merge phenodata to the phyloseq object" before advancing to the analyse steps.

Output

The analysis output consists of the following:
  • ps_nophe.Rda: A Phyloseq object without phenodata merged to the object.
  • Phenodata file (used to assign samples for statistical analysis)
  • ps_imported.txt: Text summary of the phyloseq object.
