Seurat -PCA

Description

This tool performs principal component analysis (PCA) on the highly variable genes across the single cells, selected using the tool Normalization, regression and detection of variable genes.

Parameters

  • Number of principal components to plot as heatmaps [12]
  • Number of PCs to compute [20]
  • Print loadings in a file [no]
  • Number of genes to list in the loadings file [5]

Details

As input, give the Seurat R-object (Robj) from the tool Seurat -Normalize, regression and detection of variable genes.

If you have low number of cells in your data, you might get an error message. Try lowering the number of principal components to compute.

To overcome the extensive technical noise in any single gene for scRNA-seq data, Seurat clusters cells based on their PCA scores, with each PC essentially representing a 'metagene' that combines information across a correlated gene set. Determining how many PCs to include downstream is therefore an important step, but it can be challenging. The result file PCAplots.pdf contains several plots which can help to explore the primary sources of heterogeneity in the dataset and to decide which principal components to include in the downstream analysis.

In the first plot, the top genes associated with the first two principal components are shown. Second plot shows the PCA plot: principal components 1 and 2 for the cells in the dataset, and the third plot is a heatmap focusing on principal component 1.

In heatmap , both cells and genes are sorted by their principal component scores: both extremes are shown. Rows (y-axis) represent the extreme genes, and columns (x-axis) are the extreme cells. They are "extreme" in their PCA scores, so those genes that basically best determine that particular principal component, i.e. the separation between the cells. Similarly for the cells: these cells "furthest away" (in the yellow or purple end) from each other on this spectrum of PC1. Purple: positive correlation, yellow: negative correlation. The heatmap might be handy for checking whether the genes reveal what that particular PC might be all about: for example, if the genes seem to be related to cell-cycle phase, one might want to consider regressing out that effect, or at least it's good to acknowledge this.

Next plots can be used to decide which principal components to use. By default, 12 heatmaps are plotted, focusing on the 12 first principal components, in which both cells and genes are sorted by their principal component scores. You can change the number of heatmaps to plot. Explore especially the PCs which you choose for downstream analysis: heatmaps can display the 'extremes' across both genes and cells, and can be useful to exclude PCs that may be driven primarily by ribosomal/mitochondrial or cell cycle genes.

The last plot shows the standard deviations of the principal components. Here, the cutoff is where there is an elbow in the graph.

You can also request the tool to output a txt file which lists the top N (default = 5) genes associated with high/low loadings for the first 5 PCs in the PCAloadings.txt file.

For more details, please check the Seurat tutorials.

Output

  • seurat_obj.Robj: The Seurat R-object to pass to the next Seurat tool, or to import to R. You cannot open this file in Chipster, but you can view the contents of the object using the Extract information from Seurat object tool.
  • PCAplots.pdf: Graphics presenting the principal components. Last plots are meant to aid in the selection of the PCs for the downstream analysis.
  • PCAloadings.txt: If requested, the loadings (=genes responsible) of principal components
