Retrieve data from ENA database

Description

This tool retrieves data from the European Nucleotide Archive, ENA based on an ID of a data set. The data retrieval and processing is done with the enaDataGet program.

Parameters

  • ENA ID
  • Data format
    • (Default, FASTQ, FASTA, EMBL formatted sequence) [Default]
  • Download index for CRAM files (No, Yes) [No]
  • Create a log file (No, Yes) [No]

Details

Please provide the ENA ID of the data set you wish to retrieve. As ENA stores data in many different levels and formats, the files retrieved from ENA may be in different formats depending on the data type that the ID refers to. In most cases you can use defaultÂ the format proposed by the enaDataGet program. However, for NGS reads data you should define the data format to be FASTQ. This is because for reads the enaDataGet program uses the SRA format as the default, and it is not usable in Chipster.

As the ENA data sets can be very large, downloading the data can take a long time.

Examples of output with different format options:

  • SRR16313683 with FASTQ output format: will return FASTQ formatted reads file (SRR16313683.fastq.gz)
  • ERR925793: will return CRAM formatted alignment file (16030_8#1.cram)
  • ERR925793 with FASTQ output format: will return FASTQ formatted reads file (ERR925793_1.fastq.gz)
  • AH001094: will return EMBL formatted sequence file (AH001094.dat)
  • AH001094 with FASTA output format: will return FASTA formatted sequence file (AH001094.fasta)
