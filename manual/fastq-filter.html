Filter sequences based on the number of expected errors

Description

Discards sequences which have more than the user specified number of expected errors. This tool helps to filter bad quality sequences, for example contigs made by merging paired reads with VSEARCH.

Parameters

  • Discard sequences with more than the specified number of expected errors. Decimal (0 -) [1]

Details

This tool takes a Tar package of FASTQ files, for example contigs created with tool "Combine paired reads to contigs with VSEARCH".
It filters out those sequences which have more than the user specified number of expected errors.
For more information about expected error, you can check this USEARCH manual page Expected errors predicted by Phred (Q) scores.
This page suggests to use 1 as the threshold, but it can be made stricter.
For more information about the calculations and formulas to calculate the number of expected errors, you can check this article by Robert C. Edgar and Henrik Flyvbjerg Error filtering, pair assembly and error correction for next-generation sequencing reads.

This tools is based on the fastq_filter command of VSEARCH package.

Output

The analysis output consists of the following:

  • filtered_contigs.tar: A Tar package consisting of FASTQ files containing filtered sequences. One FASTQ file per sample.
  • summary.tsv: Summary statistics, which show how many sequences have been discarded from each sample.
