Distance matrices and ordinations

Description

Produces a Euclidean or Bray-Curtis distance matrix using a phyloseq object. Can also be used to compute Aitchison distances (if selecting Euclidean distances for a CLR-transformed data set). The matrix is used to perform either an unconstrained ordination (non-metric multidimensional scaling, nMDS) or a constrained ordination (distance-based redundancy analysis, db-RDA). Further to producing an ordination plot and summary of the ordination results, a distance matrix is saved in Rda format.

Parameters

  • Type of distance measure (Euclidean or Bray-Curtis; the default is Euclidean)
  • Type of ordination (nMDS or db-RDA; the default is nMDS)
  • Phenodata variable with unique IDs for each community profile (used for data point labels)
  • Show sample IDs in ordination? (Yes, No; the default is Yes)
  • Phenodata variable for grouping ordination points by colour
  • Phenodata variable for grouping ordination points by shape
  • Phenodata variables for db-RDA formula specification (up to 5 may be specified)

Details

The tool requires the following input files:

  • A phyloseq object in Rda format
  • Data groupings are automatically fetched from a phenodata file linked to a Mothur shared file (file.opti_mcc.shared)
The tool employs the following R functions and packages:
  • ordinate(), distance() and plot_ordination() (phyloseq)
  • geom_text_repel() (ggrepel)

Output

The tool produces the following output files:

  • Text summary of ordination results (ps_ordi.txt)
  • Ordination plot (ps_ordiplot.pdf)
  • A data frame including phyloseq object sample data and a distance matrix (ps_dist.Rda)
