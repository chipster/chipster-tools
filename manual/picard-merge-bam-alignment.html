Merge aligned and unaligned BAM

Description

This Picard tool merges alignment data from a standardly aligned BAM file with data in an unmapped BAM file. Before merging, the tool also sorts the aligned BAM file by queryname (QNAME) so that merging is possible. This tool produces a new BAM file that includes all aligned and unaligned reads, while also retaining the additional read attributes from the unmapped BAM (attributes that are otherwise lost in the process of alignment). The purpose of this tool is to use information from the unmapped BAM to fix aligner output. In short, this tool can be used to add the cell and molecular barcode information, as well as other tags that may have been lost when aligning to the aligned BAM file.

After merging, Drop-seq tool TagReadWithGeneExon adds a BAM tag GE to reads which overlap an exon of a gene. This tag contains the gene name, as reported in the GTF genome annotation file. For more details, please refer to the Drop-seq manual.

Please note that this tool does not account for secondary alignments.

Please ensure that you select the same reference that was used in the alignment, and please check that the input files are assigned correctly.

For more details, please refer to the home page of Picard tools.

Parameters

  • Reference [Homo_sapiens.GRCh38] (Homo_sapiens.GRCh38, Mus_musculus.GRCm38)
  • Organism [other]
Chipster provides Ensembl-based GTF files for many organisms in the dropdown menu. If you would like to use your own GTF, please set the organism parameter to "other" and check that your GTF file has been correctly assigned in the parameter panel.

Output

  • merged.bam: Merged BAM file where reads overlapping an exon are tagged with a gene name.
