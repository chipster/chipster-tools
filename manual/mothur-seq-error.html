Assess error rates

Description

You can measure the error rate of your sequences if you have co-sequenced a mock community. This tool pulls out the sequences that were from the mock sample, calculates the overall error rate, and removes the mock sample from the fasta, count_table and sequence-taxonomy-assignment files. Error rate is defined as: sum of mismatches to reference / Sum of bases in query

Parameters

  • Mock sample name [Mock]
  • Are the reference sequences aligned (yes,no) [no]

Details

As input this tool takes a fasta file, a count_table file, a sequence-taxonomy-assignment file, and a mock reference file.
If your reference sequences are aligned, you should set the aligned parameter to yes.
This tool is based on the Mothur tools get.goups, seq.error and remove.groups. You can read more about assessing error rates in the Mothur MiSeq SOP

Output

The analysis output consists of the following files:

  • error-rate-summary.txt: Summary statictics of error rates and how many sequences were selected.
  • mock-removed.fasta.gz: Sequences with the mock sequences removed.
  • mock-removed.count_table: Count table with the mock sequences removed.
  • filtered-taxonomy-assignment.txt: Taxonomy-assignment with the mock sequences removed.
  • error-summary.tsv: Summary statistics of the remaining sequences.
