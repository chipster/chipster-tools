Assess error rates

Description

If you have co-sequenced a mock community, this tool assess the error rates. You need to give a fasta file, a count_table, a sequence-taxonomy-assignment file, and a Mock reference file. This tool pulls out the sequences that were from the “Mock” sample, specified with the "Mock group name" parameter. Then it assess the error rates and removes the Mock samples from the fasta file, count_table and sequence-taxonomy-assignment file.

Parameters

  • The Mock group name, Default "Mock"
  • Are the reference sequences aligned.(yes,no) [no]

Details

Calculates the error rates and removes the Mock sequences from the input files.
You need to give the Mock group name as parameter and the Mock reference file as input.
If your reference sequences are aligned, you should check that the aligned parameter is set to yes, otherwise to no.
This tool is based on the Mothur tools get.goups, seq.error and remove.groups.

Output

The analysis output consists of the following:

  • mock-removed.fasta.gz: Alignment with the Mock sequences removed.
  • mock-removed.count_table: Count table with the Mock groups removed.
  • filtered-taxonomy-assignment.txt: Taxonomy-assignment with the Mock groups removed.
  • error-summary.tsv: Summary statistics from remaining sequences.
  • error-rate-summary.txt: Summary statictics of error rates and how many sequences was selected.
