Seurat -SCTransform: Filter, normalize, regress and detect variable genes

Description

This tool uses the SCTransform method for normalization, scaling, and finding variable genes. You can also choose to regress out differences caused by the cell cycle stage. Before normalization, the tool filters out empty and damaged cells, as well as multiplets.
Please note, this tool and the Seurat -Filter, normalize, regress and detect variable genes tool accomplish the same goal using different methods. Feel free to select which of these two tools suits your needs best.

Parameters

  • Filter out cells which have fewer than this many genes expressed [200]
  • Filter out cells which have more than this many genes expressed [2500]
  • Filter out cells which have higher mitochondrial transcript percentage [5]
  • Number of variable genes to return [3000]
  • Regress out cell cycle differences [no]

Details

The tool filters out poor quality cells and then performs the three steps included in the SCTransform tool: normalization, scaling and finding variable genes.

Please input the Seurat R-object (Robj) generated using the Seurat setup -tool. The Seurat R-object output can be used as an input for the subsequent tool, Seurat -PCA.

Compared to the standard Seurat normalization + scaling + variable gene findings, the SCTransform workflow performs more effective normalization by efficiently removing technical effects from the data. This allows more PCs to be selected in the PCA step, as the higher PCs are more likely to represent subtle, but biologically relevant, sources of heterogeneity instead of variation in sequencing depth.

  1. Filtering is performed to remove empty and damaged cells, as well as multiplets. You can use the QC-plots.pdf to estimate the parameters for this step.

  2. Expression values are normalized using the SCTransform normalisation method, which uses Pearson residuals generated by a regularized negative binomial regression, where cellular sequencing depth is utilized as a covariate in a generalized linear model (GLM). The parameters for the model are estimated by pooling information across genes expressed at similar levels. This should remove any technical variation while also preserving biological heterogeneity. Moreover, it will also help to avoid overfitting the model to the data.

  3. Highly variable genes across the cells are detected using the Pearson residuals that were computed during the normalization step. The highly variable genes will be used in downstream analysis, like PCA.

  4. Uninteresting sources of variation in the expression values are regressed out during normalization to improve dimensionality reduction and clustering downstream. This tool regresses out the percentage of mitochondrial transcript content.
    You can also choose to regress out cell cycle differences. By choosing all differences, the tool removes all signal associated with the cell cycle. In some cases, this method can negatively impact downstream analysis, particularly in differentiating processes, where stem cells are quiescent and differentiated cells are proliferating (or vice versa). Alternatively, you can regress out the differences between the G2M and S phase scores. This means that signals separating non-cycling cells and cycling cells will be maintained while differences in the cell cycle phase amongst proliferating cells (which are often uninteresting) will be regressed out of the data. For more information, please refer to the Seurat vignette on cell cycle scoring and regression.

Output

  • seurat_obj_sctransform.Robj: The Seurat R-object to pass to the next Seurat tool, or to import into R, but not viewable in Chipster.
  • Dispersion.pdf: The variation vs. average expression plots (in the second plot, the 10 most highly variable genes are labeled). If you have selected to regress out cell cycle differences, PCA plots of cell cycle genes (before and after the regression) will be added to the end of the PDF. Also, it lists the number of highly variable genes.
