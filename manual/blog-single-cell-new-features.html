Smaller improvements include:

  • The possibility to export PCA loadings from the PCA tool . If this parameter is turned on, an .txt file listing the top N genes associatied with the loadings is given as an output. Default for the N is 5, but user can tune this.
  • Separation of the expression analysis from the clustering tool. Parameters remain the same, but separating the two tools allows faster and easier re-runs of expression analysis with different parameters.
  • Plot cells in the the order of expression. Some of the Seurat tools give as output FeaturePlots, where dots representing the cells are colored according to some feature of the data, like for example particular genes expression. In Seurat tools, by default, the cells are NOT ordered according to the expression levels, but there is a parameter that does that. We now included this parameter in the tools including this kind of plots, but it is "off" by default, according the original tool. It can be useful to turn this "on" if cells expressing given feature are getting buried.
  • Tool for removing clusters from the data. User can define either the clusters to remove, or list the clusters to keep.
  • Renaming of the clusters. By default, the cluster names are just numbers, but by importing a table with cluster new cluster names it is now possible to rename them.
  • HDF5 input file format option available for the Setup tool. The analysis can now begin from the HDF5 (.h5) files. See the new COVID data example session, and make sure you assign the files correctly in the parameters view.

Bigger improvements include:

  • SingleR cluster annotation tool. With SingleR tool and CellDex annotation datasets, users can label their cell types based on the similarity to the reference datasets. As output, users get both higher level annotations as well as more detailed labels. Some plots for estimating the quality of these annotations are also provided.
  • It is now also possible to use data normalised with the SCTransform tool in the multiple sample analysis. Note that you need to let the tool know that this normalisation method was used! When using this option, the tool acknowledges the used parameters in scaling and integration steps.
  • Last but not least, it is now possible to integrate multiple samples in the analysis, instead of just the previous two. As before, it is important that the samples are named in the Setup tool. The tool "Seurat -Find conserved cluster markers and DE genes" recognises if there are more than two samples in the object, in which case it will compare each sample to all other samples, and print out separate DEG list file for each sample. There's also a new tool, Find DE genes between chosen samples, which allows comparing any samples or group of samples to other sample/samples. See the new COVID data example session to get an overview of analysis workflow of several samples.

We further encourage our users to communicate their ideas, wishes and notes on the function of these and other tools in Chipster -help us to help you!
