Seurat -Find differentially expressed genes between clusters

Description

Find the markers for a specific cluster compared to another cluster(s) for all clusters.

Parameters

  • Find all markers [FALSE]
  • Number of the cluster of interest [1]
  • Cluster to compare to [all others]
  • Which test to use for finding marker genes [wilcox]
  • Limit testing to genes which are expressed in at least this fraction of cells [0.10]
  • Limit testing to genes which show at least this log2-fold difference [0.25]
  • Report only positive marker genes [TRUE]

Details

If the Find all markers parameter is set to TRUE, the Seurat function FindAllMarkers is used to list the markers for all clusters. In this case, each cluster is compared to all the other clusters. This parameter overwrites the selection of cluster numbers in the two parameters below. This option outputs a large table containing all the markers. If the Find all markers parameter is set to FALSE (default), the Seurat function FindMarkers is used to identify positive and negative marker genes for the clusters of interest, determined by the user. By default, differentially expressed genes are tested between the cluster of interest and all the other cells. You can also compare the cluster of interest to another cluster(s) by typing a number in the parameter field. When comparing to a group of clusters, separate the clusters by using a comma (,).

You can filter out genes prior to statistical testing by requiring that a gene has to be expressed in at least a certain fraction of cells in either of the two groups (min.pct=0.10). You can also filter out genes whose log2-fold change between the two groups is less than a given value (logfc.threshold=0.25). Both of these parameters can be set to 0, but it also needs to be accompanied with a significant increase in time, as this will test a large number of genes that are unlikely to be highly discriminatory.

Seurat currently implements the following tests:

  • "wilcox": Wilcoxon rank sum test (default)
  • "bimod": Likelihood-ratio test for single cell gene expression, (McDavid et al., Bioinformatics, 2013)
  • "roc": Standard AUC classifier
  • "t": Student's t-test
  • "tobit": Tobit-test for differential gene expression (DGE) (Trapnell et al., Nature Biotech, 2014)
  • "poisson": Likelihood ratio test assuming an underlying poisson distribution. Use only for datasets containing Unique Molecular Identifiers (UMIs)
  • "negbinom": Likelihood ratio test assuming an underlying negative binomial distribution. Use only for UMI-based datasets
  • "DESeq2": Differential expression (DE) based on a model using the negative binomial distribution (Love et al., Genome Biology, 2014). Please note that when using the DESeq2 method, you should not filter out genes prior to testing.
The Poisson, negbiom, and DESeq2 options should only be used on UMI datasets, as they assume an underlying poisson and negative-binomial distribution, respectively.

The markers.tsv results file contains marker genes and associated statistics. Please note that by default, only genes with positive log2-fold change are listed in the results file:

  • p-val = p-values for the differentially expressed genes (DEGs) (larger the p-value -> higher the likelihood that the gene is in the list is just by chance)
  • avg_log2FC = average log2-fold change (how much higher or lower the expression of this gene is in the particular cluster, compared to all other cells)
  • pct.1 = what percentage of the cells in the particular cluster show some expression for this gene
  • pct.2 = what percentage of the cells not in the particular cluster (=all the other cells) show some expression for this gene
  • p-val_adj = adjusted/corrected p-value. This value is multiple-testing corrected. When we test over thousands of genes, we would statistically start identifying some DEGs just by chance. There are different methods to correct for this, but in this tool, the Bonferroni correction is used. When filtering the table and reporting your results, please use this value.
If all markers are listed, the tool will generate a larger table, all_markers.tsv. This table contains the same values for all the cluster markers and an extra column cluster = the cluster for which this gene is a marker. Use this column to filter the table to obtain the markers for a cluster of interest using the tool Utilities / Filter table by column value. For example, to identify the markers for cluster 2, fill in the following parameters:

Column to filter by = cluster
Does the first column have a title = no
Cutoff = 2
Filtering criteria = equal-to

For more details, please refer to the Seurat tutorials.

Output

  • markers.tsv: Marker genes for the cluster of interest
  • all_markers.tsv: Marker genes for all the clusters vs. all other cells
  • top10Heatmap.pdf: Heatmap showing the expression of top ten marker genes (in terms of log-2 fold change) for each cluster.
