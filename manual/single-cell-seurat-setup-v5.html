Seurat -Setup and QC

Description

This tool creates a Seurat object from one of the following input files (see more info below):

  1. tar package of a folder which contains 10X filtered feature-barcode matrix files in MEX format
  2. Matrix/table (DGE = digital expression matrix) containing the counts per each gene in each cell (.tsv, tab separated table)
  3. 10X or CellBender filtered feature-barcode matrix in hdf5 file format (.h5)
After setting up the R-object (.Robj), some quality control plots are drawn.

Parameters

  • Project name for plotting [Project_name]
  • Keep genes which are expressed in at least this many cells [3]
  • Sample name [empty]
  • Sample group [empty]

Details

You can give one of the following files as input. Make sure that your input file is assigned correctly under the parameters:

  1. A tar packed folder of 10X feature-barcode matrix files barcodes.tsv, genes.tsv/features.tsv and matrix.mtx in MEX format. Note that you should have only these files in a single folder, the tool won't work if you have subfolders in your tar package. These three files are generated by the Cell Ranger -pipeline, and can be found under a folder corresponding to the filtered_feature_bc_matrix example here. Make sure that you use the filtered data which contains only the estimated cell-associated barcodes, as opposed to the unfiltered data containing barcodes corresponding to empty droplets in addition to cell-associated barcodes.
    You need to tar the folder into a package. There are graphical programs for this, such as 7-zip, or you can use command prompt (Windows) / Terminal (Mac) with command: tar -czvf name-for-your-package.tar.gz /path/to/folder-you-want-to-tar. You can also use the Chipster tool Utilities - Make a tar package.
  2. A table / DGE matrix which you have for example received from colleague, downloaded from some database like GEO, or generated with the DropSeq data using the tool Single cell RNA-seq - Create digital gene expression matrix in Chipster. Note, that the input file needs to be .tsv (= tab separated table) format! You can modify the file from .csv (=comma separated format) into .tsv for example in Excel (open file, choose "Save as" an "Text (Tab delimited) (*.txt)" and rename the file ending as .tsv.) Please note, that more often than not, this first step of getting the "raw" data into a format that you can analyse requires some tailoring (such as switching the comma separators etc) -you can often do this in Excel or similar tool.
  3. A 10X or CellBender filtered feature-barcode matrix in hdf5 file format (.h5). This is a general "Hierarchical Data Format version 5", designed to store and organize large amounts of data. 10X Cell Ranger software can output the feature-barcode matrix in this format in addition to the MEX format. The Remove background contamination with CellBender tool also produces the feature-barcode matrix in this format.
This tool allows you to filter genes based on the number of cells they are detected in. It also produces the following quality control plots and counts the number of cells present in the data (note that it might be easier to view the large QCplots.pdf file with an external viewer):
  • Violin plots showing the number of genes (nFeature), UMIs (nCount_RNA) and the percentage of mitochondrial genes (percent.mt) in the cells.These plots can be used to estimate the upper limits for number of genes or molecules per cell and the percentage of mitochondrial transcripts, which you can use in the filtering tool.
  • Scatter plots comparing the percent.mt and nFeature to nCount_RNA.

If you have multiple samples in your data, it is important that you use the "Sample name" parameter to name your sample (for example: covid1, covid2, normal1, normal2), and the "Sample group" parameter to determine the sample groups (for example COVID, NORMAL). These names are used later in the tools meant for multiple sample analysis, such as the CCA and integrated analysis tools. It is best not to use empty or special characters in the names.

For more details, please check the Seurat tutorials.

Output

  • seurat_obj.Robj: The Seurat R-object to pass to the next Seurat tool, or to import to R. You cannot open this file in Chipster but you can view the contents of the object using the Extract information from Seurat object tool.
  • QCplots.pdf: Quality control plots
