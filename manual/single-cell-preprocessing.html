Preprocessing DropSeq FASTQ files

Description

This tool converts a FASTQ file to an unaligned BAM files (different from a traditional BAM alignment file). It then tags the sequence reads with cell and molecular barcodes. Next, it removes reads containing low quality bases in the barcodes, trims adapters and polyA tails, and converts the preprocessed BAM file back into a FASTQ format for alignment. Then, it filters out reads that are too short from the FASTQ file. It generates an unaligned, tagged and trimmed BAM file, a trimmed FASTQ file, a summary file, and plots of the process.

Parameters

  • Base range for cell barcode [1-12]
  • Base range for molecular barcode [13-20]
  • Base quality [10]
  • Adapter sequence [AAGCAGTGGTATCAACGCAGAGTGAATGGG]
  • Mismatches in adapter [0]
  • Number of bases to check in adapter [5]
  • Mismatches in polyA [0]
  • Number of bases to check in polyA [6]
  • Minimum length of reads to keep [50]

Details

This tool combines several tools:

  • DropSeq (TagBamWithReadSequenceExtended, FilterBam)
  • Picard (FASTQ to BAM, BAM to FASTQ)
  • Trimmomatic (MINLEN)
Overview of the steps:

  1. Convert the FASTQ file into an unaligned BAM file
  2. Tag the sequence reads in the BAM file with cell and molecular barcodes
  3. Filter and trim the reads in the BAM file
  4. Convert the tagged and trimmed BAM file back to a FASTQ format to then align reads to the genome
  5. Filter out reads that are too short from the FASTQ file

Since the FASTQ format cannot hold information about the cell and molecular barcodes of the sequence reads, you will need to transform the FASTQ file into a BAM file. The BAM format has a tag field which can store barcode information. You will also need to trim and filter the sequence reads in the BAM format. However, aligners only take FASTQ format, so you will need to transform the trimmed and filtered BAM back to FASTQ format. After this preprocessing step, you will have one unaligned, tagged and trimmed BAM file holding information about the cell and molecular barcodes in the tags, as well as a FASTQ file ready for alignment. After alignment, these two files are merged using the tool Merge aligned and unaligned BAM.

Detailed description of the steps:
  1. Convert the FASTQ file into an unaligned BAM file
  2. Tag the sequence reads with cell and molecular barcodes extracts cell and molecular barcodes from the barcode reads and assigns the barcode bases in BAM tags XC and XM, respectively. This program is run once per barcode extraction to add a tag. During the first iteration, the cell barcode is extracted from the bases determined in the first parameter (default: 1-12). During the second iteration, the molecular barcode is extracted from bases determined by the base range for the molecular barcode parameter (default 13-20) of the barcode reads.
    The tool also tags reads where the base quality in the barcode drops below a certain threshold. The number of bases that fall below the threshold is marked in the XQ tag. This information is used in the subsequent filtering step.
  3. Filter and trim the reads in the BAM file performs several things:
    • XQ tags are used to filter out poor quality reads below a certain threshold (default: 10).
    • Any user determined sequences are trimmed away from the beginning of the read. The SMART Adapter sequence is offered as a default. You can select how many mismatches are allowed (default: 0), and the minimum sequence length (default: 5 bases).
    • Trailing polyA tails are hard-clipped from the reads. The tool searches for contiguous A's from the end of the read. You can determine the number of mismatches allowed (default: 0) and the minimum number of A's needed for the clipping to happen (default: 6).
  4. Convert the tagged and trimmed BAM file back to FASTQ format to then align the reads to genome
  5. Remove reads that are too short from the FASTQ file. After trimming and filtering, you might end up with some rather short reads in your BAM file. It is highly advised to remove these reads, as this accelerates the alignment step. The Trimmomatic tool and MINLEN option is used in the last step, where the default value for the Minimum length of reads to keep -parameter is set to 50.

For more details, please refer to the Drop-seq manual and the home page of Picard tools.

Output

  • [input_name].bam: Tagged, trimmed, and filtered unaligned BAM
  • [input_name].fq.gz: Trimmed and filtered FASTQ file
  • tagging_and_trimming_summary.txt: Summary of the tagging and trimming steps
  • tagging_and_trimming_histograms.pdf: Visualizations of the failed bases in the tagging steps, as well as the adapters and trimmed polyA tails
