Screen sequences for several criteria

Description

Given a fasta file and a group/count file, this tool screens out sequences that do not fulfill user-defined criteria. Note that this tool is used in twice during preprosessing (before and after aligning sequences to the reference) and different parameters apply: You should use the length parameters before alignment, and the alignment start and end parameters after the alignment.

Parameters

  • Maximum number of ambiguous bases
  • Maximum homopolymer length
  • Minimum length
  • Maximum length
  • Alignment start position (remove sequences which start after this position)
  • Alignment end position (remove sequences which end before this position)
  • Optimize by (empty, minlength, start, end) [empty]
  • Optimization criteria (0-100)

Details

Screen out sequences based on their length or alignment position. You can either choose the cut-off yourself (first four parameters), or use the optimization based on these. If you use the optimization, remember to determine the criteria: for example 85 means that Mothur will optimize the cutoff for the above chosen quality so that 85% of the sequences are kept.

After you have combined paired reads to contigs with VSEARCH, you need to run tool "Combine FASTQ or FASTA files and make a group file" to get a group and fasta file.

This tool is based on the screen.seqs command of the Mothur package.

Output

The analysis output consists of the following:

  • screened.fasta.gz: Compressed sequence file with the screened reads.
  • screened.groups: If a group file was provided as input, a group file containing the names and sample assignment of sequences after screening will be provided.
  • summary.screened.tsv: Summary statistics for the sequences.
  • screened.count: If a count file was provided as input, a count file containing the names and numbers of unique sequences after screening will be provided.
