Trim QuantSeq reads using BBDuk

Description

Given a FASTQ file containing Lexogen QuantSeq reads, removes polyA read-through, Illumina Truseq adapters, and low quality ends.

Parameters

  • Kmer length for detecting adapters and polyA [13]
  • Minimum length of kmers to report at read tips [5]
  • Trimming direction from a kmer match [Trim to the right]
  • Should read ends be trimmed based on quality [Trim both ends]
  • Threshold for quality trimming [10]
  • Minimum length for reads to be kept after trimming [20]

Details

This tool performs three steps. It starts by looking for kmer matches to two references: Illumina Truseq adapters and a 18-nt polyA sequence. You can set the kmer length and also define how short kmer matches are considered. Note that shorter matches are considered only if they occur in read ends. Once a polyA or Truseq adapter kmer match is found in a read, that kmer and all the bases to the right of it are removed (you can define the direction with a parameter).

Next the ends of the reads are scanned for low base quality and trimmed if the quality is below the Phred score given as the threshold. You can decide which end to trim.

After trimming the reads are filtered based on length.

Output

The output is a FASTQ file containing the trimmed reads.
