Seurat -Combine multiple samples

Description

This tool can be used to combine, integrate, or align multiple Seurat objects for downstream joint analysis.

Parameters

  • Normalisation method used previously [Global scaling normalization]
  • Anchor identification method [CCA, RPCA]
  • Number of CCs to use in the neighbor search [20]
  • Number of PCs to use in the anchor weighing [20]
  • Samples to use as references [No references selected]

Details

Please input the Seurat objects that were generated with Seurat Setup tool and preprocessed with Seurat -Filter, normalize, regress and detect variable genes or the SCTransform tool. Please ensure that you indicate which normalisation method was used.
Please note: You should confirm that you have assigned the sample or group name in the Setup tool (use short names like "CTRL1", "CTRL2" "TREAT1", or "TREAT2"). These names will be used downstream to separate the samples.

Normalization method: Global scaling normalization vs. SCTransform

You will need to input the tool previously used for normalization in the first parameter. If you used this tool Seurat -Filter, normalize, regress and detect variable genes, then you will need to select Global scaling normalization, which is using NormalizeData. Alternatively, if you used the tool: Seurat -SCTransform: Filter, normalize, regress and detect variable genes, you will need to use SCTransform. If selecting SCTransform, an extra step PrepSCTIntegration will be run, then FindIntegrationAnchors() and IntegrateData() functions will acknowledge the normalization method, and the ScaleData step will be skipped.

Canonical Correlation Analysis (CCA) vs. Reciprocal PCA (RPCA) in anchor identification

For the "anchor identification method" parameter, you can use the default, canonical correlation analysis (CCA), or alternatively select reciprocal PCA (RPCA), which is signficantly faster and more conservative. When determining anchors between any two datasets using RPCA, each dataset is projected into the others' PCA space and the anchors are constrained by the same mutual neighborhood requirement.

By identifying shared sources of variation between datasets, CCA is well-suited for identifying anchors when cell types are conserved, but there are substantial differences in gene expression across experiments, i.e., when experimental conditions or disease states introduce very strong expression shifts, or when integrating datasets across modalities and species. However, CCA-based integration may also lead to overcorrection, especially when a large proportion of cells are non-overlapping across datasets.

RPCA-based integration involves cells in different biological states being less likely to 'align' after integration. RPCA is therefore recommended during the following types of integrative analysis:

  • A substantial fraction of cells in one dataset have no matching type in another
  • Datasets originate from the same platform (i.e., multiple lanes of 10x Genomics)
  • There are a large number of datasets or cells to integrate
See the link to Seurat RPCA vignette below for more information.

Large dataset integration and using part of samples as references

The "Samples to use as references" parameter allows you to list sample names which are used as integration references. This is especially useful when integrating large datasets, as the tool might run out of memory if the reference samples are not predetermined. By default, all samples are compared to each other (i.e., for 10 samples, there are 45 comparisons). You can list the sample name(s) to be used as reference(s) in the corresponding parameter field, and separate multiple names using comma(s). Please ensure that the sample name is typed correctly (exactly like you typed it in the Setup tool). Also, please ensure that you select good representation of your samples as references (e.g., in the event that you may have samples from male and female patients, please pick one reference from both). Please see the link to Seurat vignette below for more information.

Integration steps and parameters

This tool performs data integration of multiple samples (in .Robj). First, Canonical correlation analysis (CCA) is performed to identify common sources of variation between the datasets. The tool then identifies anchors, assigns scores (=weights) to these anchors and then uses them to integrate the datasets.

The third parameter is used to select the dimensions from the CCA/PC which specify the neighbor search space. The neighbors are used to determine the anchors for alignment.

The fourth parameter is the number of PCs to use in the anchor weighing procedure. The anchors and their weights are used to compute the correction vectors, which are used for correction, thus allowing the datasets to be integrated.

Output

  • seurat_obj.Robj: The Seurat R-object to pass to the next Seurat tool, or to import into R, but not viewable in Chipster.
