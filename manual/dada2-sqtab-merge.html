Merge two ASV tables

Description

This tool merges two ASV/sequence tables, for example from two different sequencing runs. To merge more than two ASV tables, run the tool several times.

Details

As an input you can give two ASV tables:

  • An ASV table saved as a .Rda file, which is the output from the tool "Make an ASV table and remove chimeras". Name: seqtab_nochim.Rda
  • When merging more than two ASV tables: a merged ASV table as an .Rda object, which is produced in a previous merging step with this tool. Name: seqtab_merged.Rda

Error rate estimations can depend on the sequencing run or the sequencing machine. When analyzing data from multiple sequencing runs, it is therefore recommended to carry out sample inference independently for each run, and combine the resulting ASV tables.

When merging more than two ASV tables, first merge two ASV tables. Then, give the resulting merged ASV table as one of the inputs files.

Output

The analysis output consists of the following:

  • seqtab_merged.Rda: An object of matching the merged sequence table saved as .Rda file. It's the output of the mergeSequenceTables() function
  • summary.txt: Text file to see how many ASVs and samples there are in the merged ASV table
  • sequence_table_merged.tsv: The sequence (ASV) table produced after the mergeSequenceTables() function printed out in a .tsv file. The column names/names of each ASV sequence are being edited to asv0, asv1... to make it easier to investigate the table.
