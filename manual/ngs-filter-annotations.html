Preprocessing / Filter by column term

Description

Filters NGS data based on textual terms in the selected data column.

Parameters

  • Column to filter by (...) [empty]
  • Term to match (...) [empty]
  • Does the first column have a title (yes, no) [yes]
  • Mode (include, exclude) [exclude]

Details

Filters rows in a table based on textual terms in the selected data column. For example, you can:

  • filter for text matches such as "upstream" or "downstream" in the output file from the "Find nearest genes for regions" tool.
  • filter only genes of interest or filter out genes from a list of differentially expressed genes
  • filter based on some terms in annotation columns (like "kinase" in the "description" field)
Notice that rows where the match term is only a part of the whole term are also passing the filter.

You can choose to either include the rows with the given query terms, or to exclude them from the table.

Input

You give as an input the table you want to filter, and the query terms (such as gene names or terms) either as parameters (separate the terms with comma (,), like: "EEF2, AES, DAPK3"), or as a simple file, where each term is in one line (no headers). Note, that if the query term input file option if used, the terms given as parameters are not taken into account. Please check also that you have assigned the files correctly.

Output

A tabular text file containing only the set of rows that pass the filtering criteria.
